Install & run

macOS only. You'll need Homebrew and a microSD reader.

1 Clone

git clone https://github.com/ochias/sd-rescue.git
cd sd-rescue
chmod +x recover-sd.sh

2 Plug in your SD

Insert the microSD card. macOS may show a "disk not readable" dialog — click Ignore. Don't initialize it.

3 Run

# Standard mode — creates a full image of the SD first.
# Needs free disk space equal to the SD size.
./recover-sd.sh ~/Downloads/recovered

# Or no-image mode — reads the SD directly. No extra disk space.
./recover-sd.sh --no-image ~/Downloads/recovered

4 Inside PhotoRec

The script will print the exact buttons. Quick version:

  • Partition: pick the only one shown, or [Whole disk]
  • Filesystem type: [Other]
  • Scope: [Whole] (scans deleted regions too)
  • Destination: navigate to your folder, press C

To filter only photos, open File Opt first, press s to deselect everything, then enable jpg, tif, png, and your camera's RAW format.

FAQ

Is sd-rescue really free?

Yes. MIT-licensed, no paid tier, no upsell, no telemetry. Disk Drill, EaseUS and Stellar Photo Recovery typically scan for free but charge ~$99 to actually export recovered files. sd-rescue does the same job — using PhotoRec under the hood — for free.

Does it work for camera RAW files?

Yes. PhotoRec recovers CR2, CR3 (Canon), NEF (Nikon), ARW (Sony), RAF (Fuji), ORF (Olympus), DNG, and many others. Inside the TUI, open File Opt, press s to deselect all, then enable only the formats your camera writes.

I formatted my SD card by mistake. Can I still recover photos?

Usually yes — as long as you stop using the card immediately. Quick formats only clear the file allocation table; the actual photo bytes stay on the card until something else writes over them. Don't take new shots, don't copy anything to the card, and run sd-rescue.

Is it safe? Will it damage my SD card?

No. Both dd (when imaging) and PhotoRec open the device read-only. The script never writes back to the SD. Recovered files are saved to a folder on your Mac that you choose.

How is this different from PhotoRec on its own?

PhotoRec is the engine — sd-rescue is a guided wrapper for macOS. It installs the dependency, lists external disks, asks you to confirm the device twice (you really don't want to point recovery at the wrong drive), unmounts the SD, and either creates a dd image first or runs PhotoRec directly. Without it, the manual flow is doable but unfriendly.

Does it work on Windows or Linux?

Not directly. The script uses macOS-specific commands (diskutil, /dev/rdisk). On Linux, just use PhotoRec directly via your package manager (apt install testdisk). On Windows, use PhotoRec for Windows or QPhotoRec.

What if the SD card has bad sectors / clicks / errors?

If dd shows "Input/output error", the card is physically damaged. dd stops on errors; ddrescue retries failed reads and skips around damaged areas, so it's the right tool for that case. Open an issue if you'd like a --ddrescue flag.
